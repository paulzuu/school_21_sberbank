## О буткемпе

**Data Science Bootcamp** — это интенсив, направленный на получение и закрепление практических навыков в области науки о данных. Программа разделена на две части и охватывает широкий спектр тем: от знакомства с ИИ-агентами и основ работы в командной строке до продвинутых техник машинного обучения, визуализации данных и работы с базами данных.

<summary><strong>AI1: Личный ИИ-агент</strong></summary>

*   **Тема:** Создание персонального ИИ-агента на базе LLM с развёртыванием инфраструктуры в облаке и интеграцией с Telegram.
*   **Задания:**
    *   **Задание 1. Знакомство с инструментами:** развёртывание `n8n` в облаке Cloud.ru через Docker; подключение GigaChat API через платформу Foundation Models; сборка первого воркфлоу с узлом OpenAI Chat Model.
    *   **Задание 2. Выбор специализации:** создание Telegram-бота с AI-агентом на `n8n`; написание системного промпта, который проводит пользователя через серию вопросов (опыт, интересы, цели) и выдаёт рекомендацию по специализации; подключение `SimpleMemory` для хранения контекста диалога.
    *   **Задание 3. Дайджест новостей:** парсинг постов из Telegram-каналов по специализации через RSS; автоматическая ежедневная рассылка суммаризированного дайджеста в приватный канал с помощью GigaChat 2.
    *   **Задание 4. Анализ вакансий:** еженедельный парсинг вакансий с HH.ru через API; выделение топ-15 самых востребованных навыков с помощью AI-агента; отправка отчёта в Telegram-канал.
    *   **Задание 5. Персональный ментор:** бот, который формирует персональный контент-план по заданной теме, ищет материалы через Tavily Search и последовательно присылает посты по каждому понятию без повторов; хранение прогресса через `Static Data` в n8n.

</details>

<details>
<summary><strong>DSB10: Введение в машинное обучение</strong></summary>

*   **Тема:** Введение в основы машинного обучения с использованием библиотеки `Scikit-learn`.
*   **Задания:**
    *   `ex00–01`: Решение задачи бинарной классификации с помощью логистической регрессии, SVM и дерева решений. Визуализация границ принятия решений.
    *   `ex02`: Многоклассовая классификация, работа с категориальными признаками (One-Hot Encoding) и оценка важности признаков.
    *   `ex03`: Борьба с переобучением: разбиение на обучающую/тестовую выборки и кросс-валидация.
    *   `ex04`: Решение задачи регрессии для предсказания непрерывной величины.
    *   `ex05`: Применение кластеризации (обучение без учителя) для сегментации данных.

</details>

<details>
<summary><strong>DSB11: Продвинутые техники машинного обучения</strong></summary>

*   **Тема:** Изучение продвинутых техник машинного обучения для повышения качества моделей.
*   **Задания:**
    *   `ex00`: Применение регуляризации (L1, L2) для борьбы с переобучением.
    *   `ex01`: Автоматический подбор гиперпараметров с помощью `GridSearchCV`.
    *   `ex02`: Использование продвинутых метрик качества: `precision`, `recall`, `F1-score`, `AUC-ROC`.
    *   `ex03`: Построение ансамблей моделей: `Voting`, `Bagging`, `Stacking`.
    *   `ex04`: Создание конвейеров обработки данных и обучения (`Pipelines`) и применение ООП для структурирования ML-проекта.

</details>

<details>
<summary><strong>DSB12: Командный проект — Приложение по подбору питания</strong></summary>

*   **Тема:** Командный проект по созданию прототипа приложения для здорового питания.
*   **Описание:** Проект включал полный цикл работы Data Scientist'а:
    1.  **Сбор данных:** Парсинг информации о рецептах и их питательной ценности с использованием API.
    2.  **Исследование и ML:** Обучение моделей (регрессия и классификация) для предсказания рейтинга блюда по его ингредиентам.
    3.  **Разработка:** Создание Python-модуля и скрипта, который по списку ингредиентов предсказывает рейтинг, выводит пищевую ценность и предлагает похожие рецепты.

</details>

## Как использовать

1.  Клонируйте репозиторий:
    ```bash
    git clone https://github.com/[Your-Username]/[Your-Repo-Name].git
    ```
2.  Перейдите в директорию проекта:
    ```bash
    cd [Your-Repo-Name]/ds_bootcamp
    ```
3.  Для каждого проекта рекомендуется создать и активировать виртуальное окружение и установить зависимости:
    ```bash
    python -m venv venv
    source venv/bin/activate  # для macOS/Linux
    venv\Scripts\activate  # для Windows
    pip install -r requirements.txt
    ```
